# 配图构思协议

这份协议负责把文章观点变成读者能看懂的画面。它位于文章分析和图片生成之间，是不可跳过的决策层。

本文件主要负责概念插图和所有图型共有的命题压缩、阅读顺序与验证规则。流程图、架构图、对比图、关系图和结构图的专门规则见 `diagram-types.md`。

## 一、先决定图片为什么存在

每张图只能有一个主要任务：

1. 解释一个机制。
2. 区分两个概念。
3. 标记一次转折。
4. 让一个案例被记住。
5. 收束一种情绪。

如果一张图同时承担三项任务，通常会出现很多物件、很多箭头和很多解释，最终读者什么都记不住。

## 二、从主题降到命题

主题只是推荐系统、成长、竞争、焦虑。主题无法直接构图。

命题必须包含判断或变化：

1. 相关性推荐从相似行为推相似内容。
2. 推理推荐从行为反查真实动机。
3. 兴趣假设不是越多越好，需要证据帮助收敛。

使用句式：

```
这张图不是要展示______，而是要让读者理解______。
```

## 三、把命题变成视觉句子

视觉句子由三个部分组成：

```
主语 + 变化关系 + 结果
```

示例：

```
两张诗人内容卡 + 一条表面直线和三条向下根线 + 相似内容与真实动机的分离
```

不要先列物件再寻找含义。先确定句子，再选择最少的物件。

## 四、优先寻找主动作

最容易理解的插图通常不是名词组合，而是一个动作：

1. 掀开，适合表层与深层。
2. 穿过，适合阻隔与突破。
3. 筛掉，适合发散与收敛。
4. 对准，适合假设与证据匹配。
5. 埋下，适合积累与生长。
6. 拉近，适合距离变化和追赶。
7. 接上，适合断裂关系被重新连接。
8. 回看，适合验证、修正和复盘。

先问橙猫在做什么，再问周围需要哪些物件。一个清楚的动作通常只需要一个主物件和一个结果。

如果构思只能写成画面里有 A、B、C、D，说明还没有找到主动作。

## 五、八种关系语法

### 前后变化

适合旧方法被新方法替代、认知发生转折、状态发生变化。

画面依靠左右、前后、门槛或跨越表达。

### 表层与深层

适合行为与动机、现象与原因、结果与机制。

画面依靠地上地下、外壳内部、表面投影与根系表达。

### 分叉与选择

适合同一行为存在多种解释、不同路径或不同决策。

分叉数量控制在二至四条，不能无限发散。

### 发散与收敛

适合先提出假设，再筛选高概率方向。

关键不是画很多，而是让读者明显看见多变少。

### 阻隔与穿越

适合限制、瓶颈、旧架构约束和新通道。

画面需要明确障碍和唯一通过方式。

### 尺度与距离

适合领先与追赶、个体与时代、微小行动与巨大环境。

不能只画两个大小不同的物体，还要让距离变化服务于判断。

### 积累与生长

适合长期主义、习惯、时间和复利。

起点和结果之间必须留下过程痕迹，不能只画种子和花。

### 循环与验证

适合假设、证据、修正和再输出。

循环中每个节点必须有不同功能，不能只是四个相同图标围成圆。

## 六、阅读顺序

构思时明确三层阅读：

1. 第一眼看见唯一主语。
2. 第二眼沿着变化线移动。
3. 最后在结果或对照中理解文章判断。

如果第一眼同时出现三个同等重量的主体，构图失败。

如果读者必须从左到右逐个阅读六个小图标，插图已经退化成流程图。

如果没有变化线，只有多个相关物件，画面只是素材拼贴。

## 七、橙猫的职责

橙猫只承担三种职责：

1. 操作关键对象，例如埋种子、移动卡片、观察证据。
2. 指示阅读方向，例如看向分叉、朝向结果。
3. 提供尺度和情绪，例如面对巨大系统时显得微小。

橙猫不得承担以下职责：

1. 仅仅站在角落增加可爱感。
2. 同时操作多个对象解释完整流程。
3. 穿复杂服装扮演医生、侦探、科学家。
4. 用夸张表情代替文章判断。

生成前做角色必要性测试：暂时移除橙猫。如果画面含义和阅读方向完全不变，说明角色只是装饰，应删除橙猫或重新设计它的动作。

## 九、构思评分

每项一分，总分至少五分才可生成：

1. 单一命题是否明确。
2. 是否保留独特原文指纹。
3. 是否只有一个视觉主语。
4. 关系变化是否一眼可见。
5. 橙猫动作是否参与含义。
6. 是否能删除所有非必要元素。
7. 相似元素是否已经压缩到理解所需的最少数量。

总分至少六分才可生成。低于六分时，不写提示词，重新构思。

## 十、OneReason 构思示例

### 示例一：相关性和真实动机

```text
插入位置：
这时候推荐就不只是相似内容的延伸了。

配图任务：
区分

单一命题：
相关性只沿着表面相似继续推荐，推理会掀开行为去寻找下面的真实动机。

原文指纹：
李白、杜甫、王维内容卡，以及亲子传统文化、唐代历史、古风创作三种动机。

视觉主语：
被橙猫掀起一角的两张诗人内容卡。

关系语法：
表层与深层。

阅读顺序：
第一眼：橙猫正在掀起诗人卡片。
第二眼：桌面上有一条短线通向另一张诗人卡。
最后理解：卡片下方不是更多诗人，而是三条不同的动机根系。

橙猫动作：
用一只前爪掀起卡片边缘，低头看向下面。

一秒读图目标：
看出表面卡片下面还藏着另一层。

三秒读图目标：
看懂相似内容推荐和理解行为原因不是一回事。

删除项：
删除放大镜、箭头群、模型图标、完整流程节点和文字标签。

画面一句话：
橙猫掀起两张诗人卡片，桌面上一条短线通向第三张诗人卡，卡片下方却长出三条通向不同生活场景的根。
```

### 示例二：假设需要证据帮助收敛

```text
插入位置：
最后根据验证结果生成推荐。

配图任务：
解释

单一命题：
模型不是选择听起来最合理的解释，而是把少数假设重新对准行为证据。

原文指纹：
多种兴趣假设、用户行为序列、亲子内容和唐代历史证据。

视觉主语：
一张带有几个圆孔的假设卡。

关系语法：
循环与验证。

阅读顺序：
第一眼：橙猫拿着假设卡对准一串行为点。
第二眼：只有部分孔与行为点重合并亮起。
最后理解：不匹配的假设淡去，匹配结果才连接到推荐卡。

橙猫动作：
双爪举起一张假设卡，对准后方的行为轨迹。

一秒读图目标：
看出正在做匹配，而不是自由联想。

三秒读图目标：
看懂假设必须回到行为证据中验证，验证后才推荐。

删除项：
删除四阶段环形流程、医生隐喻、复杂分叉和多个推荐结果。

画面一句话：
橙猫把一张有孔的假设卡对准弯曲的行为点轨迹，重合的点亮起，并延伸出唯一一张推荐卡。
```
