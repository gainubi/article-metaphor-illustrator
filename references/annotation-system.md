# 手写标注系统

文字不是图片标题，而是画面结构的一部分。它负责确认读者已经看见的关系，不能替代构图。

## 一、标签的五种作用

1. **命名**：召回、粗排、精排。
2. **路径**：直接生成、回查证据。
3. **问题**：证据不足、假设过多。
4. **结果**：保留、淘汰、推荐。
5. **补充**：冷启动、长尾内容。

每个标签只能承担一种作用。

## 二、标签规划字段

每个标签在构思卡中记录：

```text
文字：
作用：
颜色：
位置：
是否可以删除：
```

如果删除标签后画面仍然清楚，而且标签没有增加专业定位，应删除。

## 三、标签数量

1. 每张图使用 1 至 5 个标签。
2. 每个标签优先 2 至 4 字，最多 6 字。
3. 不写完整句子。
4. 不放图型标题和口号。

## 四、标签位置

1. 紧贴对象或路径。
2. 与对象保持一个稳定的小间距。
3. 不横跨连接线。
4. 不放在大片留白中央。
5. 同层标签基线和字号保持接近。
6. 必要时用一条短手绘引线，不使用长箭头。

## 五、颜色语义

1. 黑色用于对象名称和普通状态。
2. 橙色用于主路径、关键动作和核心节点。
3. 红色用于问题、阻塞、淘汰和风险。
4. 蓝色用于补充状态、外部条件和次级解释。

颜色只用于语义，不用于装饰。一张图默认黑色加一种功能色。

## 六、手写感

1. 字体清楚优先于个性。
2. 轻微不规则，不能潦草。
3. 笔画粗细与主线条协调，但略细于模块轮廓。
4. 不使用儿童涂鸦体。
5. 不使用正式黑体或海报标题字。

## 七、文字 QA

1. 是否逐字正确？
2. 是否足够短？
3. 是否贴近对应对象？
4. 是否与颜色语义一致？
5. 删除后是否更清楚？
6. 是否出现图型标题或无意义口号？
7. 手机宽度下是否仍能辨认？
