# 视觉转译指南

把文字隐喻变成画面，最容易犯两个相反的错：机械直译，或者转译得和原文再无关系。这份文档讲怎么在两者之间找到平衡。

## 核心原则：保留一个名词，转译其余质感

文字隐喻由名词构成（山、雾、桥、灯），情绪藏在质感里（重、看不清、悬空、微弱但持续）。转译时不要把名词全部扔掉。先选一个最能代表原文的具体锚点保留下来，再问三个问题：

1. 这个情绪的**重量**是什么样的？（压下来的 / 飘着的 / 沉底的）
2. 它的**温度**和**光线**是什么样的？（清晨灰白 / 深夜一盏暖灯 / 正午无影）
3. 人在这个情绪里的**姿态**是什么样的？（缩着 / 站直但很小 / 走着但回头）

回答完这三个问题，用重量、光线和姿态改造这个锚点。目标不是避开原文名词，而是避免把原句逐字画成图解。

## 三层关联法

一张合格的文章插图要同时具备三层：

| 层级 | 回答的问题 | 示例 |
|---|---|---|
| 内容锚点 | 这篇文章具体讲了什么？ | 液态玻璃颜色选择器 |
| 关系结构 | 作者对它做了什么判断？ | 开源模型在具体任务上逼近闭源模型 |
| 情绪转译 | 作者希望读者感到什么？ | 兴奋，但不神化，踏实地看到希望 |

如果去掉内容锚点，画面仍然完全成立，它很可能太通用。如果只剩内容锚点，没有关系和情绪，它又会像产品截图。

## 简约的纪律

- **概念插图主体不超过两个元素**。一个角色 + 一个物，或一个物 + 一个环境暗示。第三个元素出现时，问一遍能不能删。流程图、架构图、对比图、关系图和结构图使用 `diagram-types.md` 的节点限制。
- **留白占画面一半以上**。留白不是空着，是给情绪呼吸的地方。压抑的情绪，留白压在主体上方；开阔的情绪，留白在主体前方。
- **橙白圆猫是默认角色**。文章需要动作、陪伴、方向或尺度时，使用固定参考图中的橙白圆猫。它不是装饰物，动作必须服务于原文。
- **角色宜小不宜大**。默认占画面约 6% 至 10%，只负责提供动作、方向和尺度。大部分画面仍然交给原文指纹、关系结构和留白。
- **外形保持稳定**。圆胖团子体型、橙色头背、奶白脸腹和前爪、小三角耳、黑色椭圆眼睛与小鼻子不能漂移。允许改变姿势，不允许变成写实猫、柴犬、狐狸或普通卡通猫。
- **线条是主角**。使用清晰、有力度的墨线，轮廓线略粗，内部细节线更细。允许轻微手绘抖动，但不要虚弱、毛糙或反复描摹。每个元素尽量用少量线条说清楚。
- **颜色只做点睛**。以奶白纸面和深墨线为主，只保留一处小面积暖色作为画面的心跳。不要用多块水彩铺色替代构图，也不要依靠渐变和复杂阴影制造氛围。
- **减少装饰纹理**。纸面可以有极轻微自然质感，但不能出现明显颗粒、大片铅笔排线、密集阴影或装饰性笔触。
- **领域文章要保留领域证据**。科技文章至少保留一个技术对象或操作痕迹，商业文章至少保留一个交易或组织线索，生活文章至少保留一个具体动作或旧物。不要用船、路、山等通用意象替换所有领域内容。

## 正反例

**例一：文章写"中年转行像船到江心"**

- 直译（差）：一条船在江中央，船上一个人愁眉苦脸。太图解，像课本插图。
- 转译（好）：极简水面占满画面下方三分之一，一个很小的背影坐在一只小船的船头，船头朝向画面外，远处什么都没有。留白即江面。情绪：悬而未决，但人是朝前的。

**例二：文章写"父亲老了，像一盏越来越暗的灯"**

- 直译（差）：一个老人旁边放一盏暗灯。两个元素互相解释，反而都弱了。
- 转译（好）：一盏旧台灯在画面右下角，光晕很小但是全画唯一的暖色，光晕边缘有一副老花镜。人不出现。情绪：微弱、固执、还在。

**例三：文章写"AI 时代的焦虑，像所有人都在跑而你在系鞋带"**

- 直译（差）：一群人跑步，一个人蹲着系鞋带。太字面，幽默感盖过了焦虑。
- 转译（好）：画面大部分是密集的、模糊的、向同一方向倾斜的细线条（暗示人流和速度），左下角一个清晰的、静止的小人蹲着。清晰与模糊的对比就是情绪本身。

**例四：文章写"一个开源 Coding 模型做出了闭源模型没做好的液态玻璃颜色选择器"**

- 脱文（差）：两只船一前一后，象征闭源领先、开源追赶。判断是对的，但看不出文章在讲 Coding，也看不出关键 Case。
- 直译（差）：完整复刻软件截图，加上多个模型 Logo 和排行榜。信息很多，却失去插图的余味。
- 转译（好）：一个小型代码窗口旁悬浮着半透明的液态玻璃色盘，彩色光线在玻璃边缘发生折射。远处还有一个更大但较淡的玻璃面板。保留 Coding 和 Liquid Glass 两个原文指纹，用一近一远表达具体任务里的追赶关系，用暖光表达兴奋和希望。

## 脱文测试

生成前后都问一次：

1. 熟悉文章的人能否看出这是哪一个 Case 或哪一段？
2. 画面里是否有至少一个只能来自这篇文章的线索？
3. 如果把插图挪到另一篇同主题文章里，它是否仍然毫无违和？

第 3 题如果回答是，说明画面太泛，需要补回原文指纹。

## 几种常见情绪的画面语汇

| 情绪 | 倾向的画面语言 |
|---|---|
| 孤独但平静 | 小人物 + 大留白，光线均匀，无阴影压迫 |
| 压抑 | 主体上方留白少、色块下压、线条变密 |
| 释然 / 收尾 | 主体面向画面外或远方，前方大片留白 |
| 怀旧 | 一件旧物的特写，线条略抖，纸张质感重 |
| 笨拙的勇敢 | 小人物姿态前倾，环境元素略大于常理 |
| 时代感 / 洪流 | 重复性线条或元素的阵列，其中一个不同 |

## 线条检查

1. 缩小图片后，主体轮廓是否仍然一眼可辨？
2. 轮廓线是否明确，内部线条是否比轮廓更轻？
3. 是否能删掉一半装饰线而不影响含义？
4. 色彩全部去掉后，构图是否仍然成立？
5. 是否出现大面积水彩晕染、铅笔涂抹或密集排线？出现就需要简化。

## 角色造型检查

固定角色是高语义元素。模型容易在多次生成中改变体型和物种，因此生成前要逐项限定：

1. 是否读取了 `assets/orange-cat-main-visual.png` 作为角色参考？
2. 圆胖体型和橙白色块是否保持稳定？
3. 姿态是否表达原文动作，而不是为了卖萌？
4. 是否只使用一个必要的小道具？
5. 角色是否控制在约 6% 至 10%，让原文意象和空间关系成为真正主体？

这张表是起点不是终点，遇到表里没有的情绪，回到三个问题（重量、光线、姿态）现场推。
